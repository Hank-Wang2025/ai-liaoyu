# 统一入口仅保留开始疗愈设计

## 目标

简化操作员入口流程，让根页面只支持开始新的评估。

系统应满足：

- 保留 `/` 作为统一入口页
- 完全移除“继续上次疗愈”能力
- 首页默认直接展示评估流程，不再需要额外点击展开
- 疗愈中关闭客户端、刷新页面或崩溃后，不再恢复未完成会话，而是回到起始流程

## 产品决策

- 保留单一入口页 `/`
- 首页继续保留欢迎品牌区、副标题、说明文案和语言切换
- 移除“继续上次疗愈”卡片以及所有恢复相关提示
- “开始新评估”默认直接展示，不再额外折叠/展开
- 客户端关闭、刷新或崩溃后，不保留未完成疗愈会话
- 保留 `/assessment` 作为兼容路由，继续渲染相同的评估体验

## 变更原因

此前确认过的“开始/继续”方案，引入了客户端恢复状态、后端校验和第二条入口路径。

当前部署目标不再需要这部分能力。移除恢复功能后，操作路径更少，陈旧会话边界更少，也更符合按摩椅现场使用场景：打开系统后直接开始新一轮疗愈。

## 选定架构

保留已经完成的评估面板抽取，但把入口壳简化回“仅开始疗愈”。

### 首页

`WelcomePage.vue` 继续作为根入口页，但页面内容调整为：

- 欢迎/品牌头部
- 欢迎副标题
- 简短说明文案
- 语言切换
- 直接显示在主体区域中的评估面板

页面中不再存在继续疗愈卡片、恢复失败提示或无可恢复会话提示。

### 评估流程

保留 `AssessmentEntryPanel.vue`，继续作为以下流程的独立承载组件：

- 语音采集
- 麦克风不可用时切换到手动情绪按钮
- 情绪评估结果展示
- 推荐方案展示
- 开始疗愈动作

这样可以继续保持评估逻辑独立，避免再次把逻辑塞回一个过大的页面文件。

### 会话模型

客户端会话恢复为纯内存态。

前端不再：

- 存储可恢复疗愈快照
- 调用后端做恢复校验
- 在客户端重启后恢复上一次疗愈状态

如果疗愈过程中客户端被关闭或刷新，新的一次应用启动应直接从首页开始。

### 疗愈页行为

`TherapyPage.vue` 只支持从新评估开始后进入。

如果页面加载时不存在有效的内存态会话，继续回跳到 `/`。

所有恢复专用的阶段/时长回填逻辑都应移除。

## 前端设计

### 入口页结构

首页应呈现一条清晰的开始流程：

1. 欢迎头部
2. 欢迎副标题
3. 简短说明文案
4. 评估区域标题
5. 默认已展开的评估界面
6. 语言切换

操作员不需要额外点击一次，才能看到评估控件。

### 兼容路由

`/assessment` 继续保留给旧的内部跳转使用，并通过薄包装页继续渲染同一个 `AssessmentEntryPanel`。

它不需要复制首页完整的欢迎品牌壳。真正的主入口仍然是 `/`，而 `/assessment` 只是兼容性直达评估入口。

## 清理范围

移除此前“继续疗愈”方向引入的恢复相关代码，包括：

- `ResumeSessionCard.vue`
- `sessionResume.ts`
- resume snapshot 相关类型
- store 中所有恢复快照状态和辅助方法
- `sessionApi.getSession`
- 恢复相关文案键
- 恢复相关测试
- 疗愈页中的恢复进度回填逻辑

同时在迁移时清理历史 `therapy_resume_snapshot` 本地存储键，避免旧数据在功能移除后继续残留。

## 后端范围说明

本次变更移除的是客户端产品流中的恢复能力，不是删除所有后端 session 接口。

- 删除仅为恢复校验新增的前端 `sessionApi.getSession`
- 删除所有前端恢复校验与恢复回填行为
- 通用后端 session/report 接口可以保留，除非后续确认没有其他功能再使用它们

面向操作员的结果要求是：客户端流程中不再存在任何“继续疗愈”功能。

## 错误处理

- 麦克风不可用和手动情绪兜底行为保持不变
- 首页不再出现任何恢复相关错误提示
- 疗愈页刷新后，因为没有有效内存会话，应安全回到 `/`
- 历史 `therapy_resume_snapshot` 数据不能导致启动异常，并应被安全清理

## 非目标

- 不恢复旧的“欢迎页 -> 评估页”两步流程
- 不再增加任何后端恢复会话能力
- 不改造疗愈/报告主流程，除非是移除恢复逻辑所必需
- 不做与本次入口和会话清理无关的额外重构

## 验证

- 首页只保留开始评估流程的回归测试
- `/assessment` 兼容路由仍能渲染评估面板的回归测试
- 评估提取组件仍完整承载手动兜底流程的回归测试
- 疗愈页不再依赖恢复回填逻辑的回归测试
- session store 不再引用恢复快照辅助函数的回归测试
- 旧 `therapy_resume_snapshot` 数据会被忽略或安全清理的回归测试
- 手工验证在 `/therapy` 刷新后会安全回到 `/`，不会恢复会话
- `cmd /c npx vue-tsc --noEmit`
- 手工验证开发模式下 `/` 会直接进入评估体验
