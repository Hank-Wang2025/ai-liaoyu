# Codex Windows 入门培训课件设计

**日期：** 2026-03-17

## 目标

产出一份面向第一次接触 Codex 的同事的 PowerPoint 培训课件，主题为 `Codex 的安装与使用说明`。课件以 Windows 场景为主，时长控制在 20-30 分钟，文案语气采用中性技术说明书风格。

## 受众与边界

- 受众：第一次接触 Codex 的内部同事
- 平台：Windows 为主
- 使用场景：培训讲解与会后自学
- 内容边界：聚焦安装、首次登录、基本使用、常见工作流、风险提示与排障
- 不展开的内容：深度高级配置、团队级治理策略、跨平台完整差异对比

## 内容口径

课件以当前官方资料和本机已安装 CLI 信息为准：

- 官方安装建议包含 `npm install -g @openai/codex`
- 官方说明要求 `Node.js 22+`
- Windows 原生 CLI 当前仍标注为实验性支持，长期稳定使用建议配合 WSL；同时官方已有 Windows App 可供新手作为替代入口
- 本机 `2026-03-17` 实测 CLI 版本为 `codex-cli 0.114.0`

## 课件结构

共 11 页：

1. 封面
2. Codex 是什么
3. Codex 能帮我们做什么
4. 使用前准备
5. Windows 安装流程
6. 首次启动与登录
7. 基本使用方式
8. 典型工作流示例
9. 常见问题与排障
10. 使用规范与最佳实践
11. 结尾与 Q&A

## 设计原则

- 培训课件型结构，而不是逐步操作手册
- 深色封面与结尾页，浅色内容页
- 技术风格配色：深蓝为主，青绿为强调色
- 每页至少有一个视觉元素：流程图、卡片、命令框、步骤条或信息图
- 页面可投屏阅读，正文不堆砌大段文字

## 单页内容要求

### 1. 封面

- 标题：`Codex 安装与使用说明`
- 副标题：`Windows 新同事入门培训`
- 视觉元素：终端窗口风格图形与抽象网格背景

### 2. Codex 是什么

- 定义 Codex 是命令行中的 AI 编码助手
- 说明其核心能力：理解仓库、修改代码、执行命令、协助排障与审查
- 使用简短对比说明 Codex 与普通聊天式问答工具的区别

### 3. Codex 能帮我们做什么

- 用 4 张能力卡片展示典型场景
- 场景包含：阅读仓库、实现小改动、生成测试、排查错误或做 code review

### 4. 使用前准备

- 罗列 Windows 环境前提
- 包括：Node.js、npm、终端、网络条件、OpenAI 账号/API Key
- 增加兼容性提示：原生 Windows CLI 可用，但官方仍建议长期使用时考虑 WSL

### 5. Windows 安装流程

- 采用流程图或步骤卡片
- 包含：安装 Node.js、校验 `node --version` 与 `npm --version`、执行安装命令、校验 `codex --version`
- 明确推荐命令：`npm install -g @openai/codex`

### 6. 首次启动与登录

- 展示 `codex` 与 `codex login`
- 说明设备认证/API Key 两类入口
- 解释首次进入时常见的工作目录、审批和沙箱概念

### 7. 基本使用方式

- 交互式对话：直接运行 `codex`
- 非交互执行：`codex exec`
- 在仓库中运行时的典型行为：读文件、提出计划、执行命令、修改文件

### 8. 典型工作流示例

- 用一个对新手友好的实例串起来
- 示例建议：让 Codex 阅读仓库并修改一个页面文案或补一个小测试
- 使用流程为：描述目标 -> 审阅计划 -> 查看 diff -> 验证结果

### 9. 常见问题与排障

- 命令不存在
- 登录失败
- 权限审批阻塞
- Windows 原生环境兼容问题
- 版本或 PATH 异常

### 10. 使用规范与最佳实践

- 从小任务开始
- 高风险命令先确认
- 审阅改动再接受
- 对生成结果做验证
- 不把敏感信息直接粘贴进提示词

### 11. 结尾与 Q&A

- 一句话收束：先安装，再从低风险小任务开始协作
- 预留答疑空间
